Description

A new program starts with a new repository, where you keep the code and history of changes. You'll create a repository with one file. You don't need to know the programming language of the given program. All you should focus on now is the git commands.

Operate within the repository folder and execute the stage commands there. Depending on your platform, you might encounter a problem that says, "No tests have been run." To fix this, you might have to activate Git from the command line and third-party software.

The following code snippets are here to help you reach the objectives for a project stage. Please follow the objectives carefully in order to pass the stage.

if __name__ == "__main__":
    print("Helo, world!")

Objectives

  • Initialize an empty git repository inside the repository folder;

  • Create a Python file named main.py and copy the code block above into it;

  • Add the changes to the staging area;

  • Commit the changes with the message Hello, world.

There some typos in the initial version of our program. Have you ever heard of bugs? However, even if you want to fix these typos right now, be patient, you'll fix them later.

You can use the git config --global init.defaultBranch main command to set main as the default branch in your system settings.

Tip: If you want to start over, delete the .git directory.
