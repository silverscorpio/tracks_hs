Description

While working on the newer version of the program, you can still make urgent changes to the main branch if needed. As you might remember, we had some typos in the main branch. The work on the develop branch may last a long time, so let's make a hotfix for this problem.

Start the task by creating a new hotfix branch from the main. Alter the content of the main.py file again, when you finish the changes commit them and merge to the main branch.

if __name__ == "__main__":
    print("Hello, world!")
Objectives
  • Switch to the main branch;
  • Create a new branch with the name hotfix/no-more-typos;
  • Modify the main.py file to be the same as the code block above;
  • Add changes to the staging area;
  • Commit the changes with the message Fix typos;
  • Switch to the main branch;
  • Merge hotfix/no-more-typos branch to the main.

Now you will have a good clean version of your code in the main branch.
